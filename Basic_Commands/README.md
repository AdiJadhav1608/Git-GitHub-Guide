## ğŸ“ Tracking Changes

| Command | Description |
|----------|-------------|
| `git status` | Shows the current state of your working directory and staging area. |
| `git add <file>` | Stages a specific file for commit. |
| `git add .` | Stages all modified files. |
| `git diff` | Displays differences between working files and the last commit. |

---

## ğŸ’¾ Committing Changes

| Command | Description |
|----------|-------------|
| `git commit -m "message"` | Commits staged changes with a descriptive message. |
| `git log` | Displays a list of previous commits. |
| `git show <commit>` | Shows detailed information about a specific commit. |

---

## ğŸ§± File Management

| Command | Description |
|----------|-------------|
| `git mv <old> <new>` | Renames or moves a file. |
| `git rm <file>` | Removes a file from the repository. |

---

## ğŸ§¹ Undoing Changes

| Command | Description |
|----------|-------------|
| `git restore <file>` | Discards changes in a file (restores last committed version). |
| `git reset <file>` | Unstages a file without deleting its content. |
| `git checkout -- <file>` | Reverts file changes to the last commit. |

---

## ğŸ§­ Viewing History

| Command | Description |
|----------|-------------|
| `git log --oneline` | Shows commits in one-line format. |

| `git blame <file>` | Shows which commit and author last modified each line of a file. |
